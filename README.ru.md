# Componenta Profiler

Профилировщик времени разработки для жизненного цикла запроса, времени загрузки, инструментирования сервисов и отображения временной шкалы.

Используйте пакет как необязательную зависимость для разработки. Его легко отключить в боевом окружении: код приложения может зависеть от `ProfilerInterface`, а конфигурация решает, подключать реальный профилировщик или пустую реализацию.

## Установка

```bash
composer require componenta/profiler
```

Пакет намеренно не объявляет `extra.componenta.config-providers`.
Регистрируйте `Componenta\Profiler\ConfigProvider` вручную из конфигурации приложения только для разработки, когда профилирование включено.

## Требования

- PHP 8.4+

## Связанные пакеты

| Пакет | Зачем нужен здесь |
|---|---|
| `psr/http-message` / `psr/http-server-middleware` | `TimingMiddleware` и `ProfilerMiddleware` работают с PSR-7 запросами, PSR-7 ответами и PSR-15 обработчиками. |
| `componenta/pipeline` | Профилирование может оборачивать конвейер промежуточных обработчиков. |
| `componenta/app` | Стартовые загрузчики и ранние отметки относятся к жизненному циклу приложения. |
| `componenta/di` | Делегаторы сервисов могут включать профилирование выбранных классов. |

## Основной API

```php
use Componenta\Profiler\Profiler;

$profiler = new Profiler();

$profiler->mark('boot');

$span = $profiler->span('controller');
// run work
$span->close();

$marks = $profiler->marks();
```

Отметки являются упорядоченными записями с метками, временем, использованием памяти и типом:

- `Point`: мгновенная отметка;
- `Begin`: начало интервала;
- `End`: конец интервала.

## Ранние отметки загрузки

`Marks` — глобальный для процесса фасад для точек, которые происходят до появления DI-контейнера:

```php
use Componenta\Profiler\Marks;

Marks::mark('autoload.loaded');
```

Интеграционный слой приложения может слить эти ранние отметки в настроенный `ProfilerInterface`.

## Отображение

Рендереры преобразуют собранные отметки в форматы для отладки:

- `JsonRenderer`: структурированный машиночитаемый вывод;
- `ServerTimingRenderer`: значение заголовка `Server-Timing`;
- `WaterfallRenderer`: человекочитаемая временная шкала;
- `CompositeRenderer`: выбирает рендерер.

## HTTP и интеграция с фреймворком

Интеграционный слой включает:

- `TimingMiddleware`: отмечает обработку PSR-15 запроса;
- `ProfilerMiddleware`: отдает вывод профилировщика, когда совпал триггер;
- `TimingBootloader`: записывает интервалы работы стартового загрузчика;
- делегаторы сервисов для профилирования выбранных классов;
- `ProfilingPipelineFactory` для оборачивания конвейеров промежуточных обработчиков.

## Триггеры

Триггеры решают, нужно ли включить вывод профилировщика для запроса:

- триггер по параметру строки запроса;
- триггер по заголовку;
- триггер по cookie;
- составной триггер.

Так профилирование остается включаемым по запросу и не раскрывает данные профилировщика по умолчанию.

## DI-регистрация

`ConfigProvider` регистрирует сервисы профилировщика, рендереры, триггеры и необязательные точки интеграции. Держите пакет зависимостью только для разработки, если у приложения нет явного сценария наблюдаемости в боевом окружении.
